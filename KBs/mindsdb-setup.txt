================================================================================
STEP-BY-STEP MCP SERVER CONFIGURATION GUIDE
================================================================================

Step 1: Start MindsDB Docker Container with Authentication
----------------------------------------------------------
docker run -d --name mindsdb \
  -e MINDSDB_USERNAME=admin \
  -e MINDSDB_PASSWORD=YOUR_MINDSDB_PASSWORD \
  -p 47334:47334 \
  -p 47335:47335 \
  mindsdb/mindsdb

Step 2: Wait for MindsDB to Start (30 seconds)
----------------------------------------------
sleep 30
curl -s http://localhost:47334/api/status

Expected output:
{"mindsdb_version":"25.12.0","environment":"local","auth":{"confirmed":false,"http_auth_enabled":true,"provider":"local"}}

Step 3: Get Authentication Token
--------------------------------
curl -s -X POST -d '{"username":"admin","password":"mindsdb123"}' \
  -H "Content-Type: application/json" \
  http://localhost:47334/api/login

Expected output:
{"token":"pat_XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX"}

Save this token for the next step.

Step 4: Add MindsDB MCP Server to Claude CLI
--------------------------------------------
claude mcp add mindsdb-local http://localhost:47334/mcp/sse \
  --transport sse --scope user \
  --header "Authorization: Bearer YOUR_TOKEN_FROM_STEP_3"

Example with actual token:
claude mcp add mindsdb-local http://localhost:47334/mcp/sse \
  --transport sse --scope user \
  --header "Authorization: Bearer YOUR_AUTH_TOKEN_HERE"

Step 5: Verify MCP Connection
-----------------------------
claude mcp list

Expected output:
✓ mindsdb-local: http://localhost:47334/mcp/sse (SSE) - Connected

Step 6: (Optional) Remove and Reconfigure MCP Server
----------------------------------------------------
If you need to update the token or reconfigure:

# Remove existing config
claude mcp remove mindsdb-local

# Add with new token
claude mcp add mindsdb-local http://localhost:47334/mcp/sse \
  --transport sse --scope user \
  --header "Authorization: Bearer NEW_TOKEN"

docker start mindsdb

# Stop MindsDB container
docker stop mindsdb

# View container logs
docker logs mindsdb

# Restart container
docker restart mindsdb

# Remove container (data will be lost)
docker rm -f mindsdb

# Check container status
docker ps --filter "name=mindsdb"

================================================================================
TROUBLESHOOTING
================================================================================

Issue: MCP connection fails after some time
Solution: Token expired. Regenerate token using Step 3 and update MCP config.

Issue: Container won't start
Solution: Check if port 47334 is already in use:
  lsof -i :47334
  # Kill the process or use different port

Issue: Authentication fails
Solution: Verify username/password match what was set in docker run command.
