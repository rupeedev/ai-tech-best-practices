STEP-BY-STEP CONFIGURATION GUIDE
================================================================================

Step 1: Clone the Graphiti Repository
--------------------------------------
cd /tmp
git clone --depth 1 https://github.com/getzep/graphiti.git graphiti-mcp

Step 2: Create Environment Configuration File
----------------------------------------------
Create .env file at /tmp/graphiti-mcp/mcp_server/.env with:

# Graphiti MCP Server Environment Configuration

# OpenAI API Configuration (REQUIRED)
OPENAI_API_KEY=your_openai_api_key_here
MODEL_NAME=gpt-4o-mini

# FalkorDB Configuration (bundled in Docker)
FALKORDB_DATABASE=graphiti_db
GRAPHITI_GROUP_ID=main

# Concurrency Control (adjust based on your OpenAI tier)
SEMAPHORE_LIMIT=10

Step 3: Start the Graphiti MCP Server with Docker
-------------------------------------------------
cd /tmp/graphiti-mcp/mcp_server/docker
docker compose -f docker-compose.yml up -d

Step 4: Verify Container is Running
-----------------------------------
docker ps --filter "name=graphiti"

Expected output shows container running on ports:
- 8000 (MCP Server)
- 6379 (FalkorDB)
- 3000 (FalkorDB UI)

Step 5: Add Graphiti MCP to Claude CLI
--------------------------------------
claude mcp add graphiti http://localhost:8000/mcp/ --transport http --scope user

Step 6: Verify MCP Connection
-----------------------------
claude mcp list

Expected output:
✓ graphiti: http://localhost:8000/mcp/ (HTTP) - Connected

  Available MCP Tools                                                                                                                   
                                                                                                                                        
  Graphiti provides these knowledge graph tools:                                                                                        
                                                                                                                                        
  1. add_episode - Add new information to the knowledge graph                                                                           
  2. search_nodes - Search for entities in the graph                                                                                    
  3. search_edges - Search for relationships between entities                                                                           
  4. get_entity_edge - Get edges for a specific entity                                                                                  
  5. delete_entity_edge - Remove entity or relationship                                                                                 
  6. clear_graph - Clear the entire knowledge graph
